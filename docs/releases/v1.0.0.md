# GuardianStar v1.0.0

## Summary

`v1.0.0` marks the first fully demoable GuardianStar release.  
This version upgrades the project from a single-device prototype into a multi-device monitoring flow with automated builds and backend tests.

## Highlights

- Multi-device backend state keyed by `deviceId`
- Device switcher in the Monitor app
- Per-device safe-zone assignment and removal
- Client-side safe-zone sync and Android Geofence registration
- Alert propagation from Client to backend to Monitor
- Persistent backend state via `server_state.json`
- Docker packaging for the backend
- GitHub Actions for tests, APK builds, artifact uploads, and tag releases

## Included Components

### Client app

- Foreground location tracking
- Runtime permission flow
- Server address configuration
- Safe-zone synchronization per device

### Monitor app

- Device list and device switching
- Live location view
- History polyline rendering
- Safe-zone controls
- Alert display

### Backend

- Health endpoint
- Device list endpoint
- Per-device latest location, history, alerts, and safe zone
- Basic persistence across restarts

## Validation Performed

- `python -m unittest discover -s .\Client\server -p "test_*.py"`
- `.\gradlew.bat clean assembleDebug :monitor:assembleDebug`

## Known Gaps

This release is still a prototype and does not yet include:

- login and guardian-child account binding
- database-backed storage
- push notifications
- production API authentication
- hardened secret handling for map keys

## Recommended Next Step

The most valuable follow-up after `v1.0.0` is:

1. move backend state from JSON to SQLite or PostgreSQL
2. add user accounts and device ownership rules
3. wire alert push delivery to the guardian side
