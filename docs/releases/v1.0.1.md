# GuardianStar v1.0.1

## Summary

`v1.0.1` focuses on project maturity and open-source readiness.

This release upgrades repository presentation, governance, and release hygiene while preserving the existing working feature set introduced in `v1.0.0`.

## Highlights

- Upgraded README to open-source project homepage style
- Added one-diagram system architecture page
- Replaced screenshot placeholders with unified real-device style visuals
- Added contribution, security, and code-of-conduct policies
- Added issue templates and pull request template
- Added deployment guide and maintainable changelog
- Added dependabot configuration for regular dependency updates
- Kept CI pipeline and build/release workflow intact

## Technical Changes Included

### Backend Hardening

- Switched runtime server to `ThreadingHTTPServer`
- Added stricter payload validation for:
  - location coordinates
  - safe-zone radius (> 0)
- Expanded backend test coverage for invalid request handling

### Build Consistency

- Aligned Android Gradle plugin version to `8.2.2` across project modules

## Documentation Additions

- `docs/architecture.md`
- `docs/DEPLOYMENT.md`
- `CHANGELOG.md`
- `CONTRIBUTING.md`
- `SECURITY.md`
- `CODE_OF_CONDUCT.md`

## Validation Performed

- `python -m unittest discover -s .\Client\server -p "test_*.py"`
- `.\gradlew.bat --no-daemon clean assembleDebug :monitor:assembleDebug`

## Upgrade Notes

No runtime migration is required from `v1.0.0` to `v1.0.1`.

This release is backward compatible at API level for current prototype endpoints.
